# Mini-Coze 项目架构说明

## 一、项目概述

Mini-Coze 是一个轻量级的 AI 工作流编排工具，类似于 Dify 和 Coze 的简化版本。它允许用户通过可视化界面拖拽节点来构建 AI 应用工作流，实现知识库检索、AI 对话等功能。

### 1.1 核心功能

- **可视化工作流编排**：拖拽式节点编辑器，支持 5 种节点类型
- **知识库 RAG 检索**：支持 PDF、Word、TXT 文档上传和混合检索
- **AI 对话**：集成 DeepSeek 等大模型，支持流式输出
- **工作流执行**：拓扑排序执行引擎，支持条件分支

### 1.2 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue 3 + TypeScript + Vite | 现代化响应式框架 |
| 前端状态管理 | Pinia | Vue 官方推荐状态管理方案 |
| 工作流可视化 | @vue-flow/core | Vue 工作流编辑器 |
| 后端 | NestJS + TypeScript | 企业级 Node.js 框架 |
| ORM | TypeORM | 支持多种数据库 |
| 数据库 | SQLite (开发) / PostgreSQL (生产) | 轻量与性能兼顾 |
| AI 调用 | OpenAI SDK | 兼容多家国内大模型 |

## 三、核心模块详解

### 3.1 工作流引擎（Workflow Engine）

位置：`backend/src/workflow/executor.service.ts`

#### 节点类型

```typescript
enum NodeType {
    INPUT = 'INPUT',                    // 开始节点 - 接收用户输入
    AI_AGENT = 'AI_AGENT',              // AI 节点 - 调用大模型
    CONDITION = 'CONDITION',            // 条件节点 - 分支判断
    KNOWLEDGE_RETRIEVAL = 'KNOWLEDGE_RETRIEVAL',  // 知识检索节点
    OUTPUT = 'OUTPUT',                  // 结束节点 - 输出结果
}
```

#### 执行原理

工作流引擎使用**拓扑排序算法**执行节点：

1. **构建执行图**：根据节点和边的连接关系构建有向图
2. **入度计算**：统计每个节点的入边数量
3. **顺序执行**：从入度为 0 的节点开始，依次执行
4. **数据传递**：上游节点的输出作为下游节点的输入

```
示例工作流执行顺序：

开始(INPUT) → 知识检索(KNOWLEDGE_RETRIEVAL) → AI回答(AI_AGENT) → 结束(OUTPUT)
     ↓                    ↓                         ↓              ↓
  用户输入           检索文档片段            生成AI回答       最终结果
```

### 3.2 知识库系统（Knowledge Base）

#### 文档处理流水线

```
用户上传文档
    ↓
文档解析 (pdf-parse / mammoth)     # 提取文本
    ↓
文本分片 (ChunkingService)         # 切分成小块
    ↓
生成向量 (EmbeddingService)        # 转换为向量
    ↓
存入数据库                         # 保存片段和向量
```

#### 分片策略

位置：`backend/src/knowledge/services/chunking.service.ts`

支持三种分片方式：

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| fixed | 固定长度切分 | 简单场景 |
| semantic | 按段落分割，保留语义 | 推荐方案 |
| recursive | 递归分割 | 复杂文档 |

默认配置：
- 分片大小：1000 字符
- 重叠长度：200 字符

#### 检索算法

位置：`backend/src/knowledge/services/search.service.ts`

采用**混合检索（Hybrid Search）**：

```
最终得分 = 0.3 × BM25分数 + 0.7 × 向量相似度
```

- **BM25**：基于词频的经典检索算法，适合关键词匹配
- **向量相似度**：基于语义的理解，适合概念匹配

### 3.3 前端工作流编辑器

位置：`frontend/src/components/FlowCanvas.vue`

#### 功能特性

- **拖拽添加节点**：从工具栏拖拽节点到画布
- **连线**：拖拽连接节点，构建执行流程
- **撤销/重做**：最多 50 步历史记录
- **工作流验证**：运行前检查配置完整性
- **实时日志**：SSE 流式显示执行过程
- **结果预览**：双击节点查看运行结果

## 六、使用指南

### 6.1 启动项目

```bash
# 1. 启动后端
cd backend
npm run start:dev

# 2. 启动前端
cd frontend
npm run dev

# 3. 访问
打开浏览器访问 http://localhost:5174
```

### 6.2 创建工作流

1. 点击「新建工作流」
2. 从左侧工具栏拖拽节点到画布
3. 连接节点构建流程
4. 配置每个节点的参数
5. 点击「运行」测试

### 6.3 使用知识库

1. 进入「知识库」页面
2. 创建知识库
3. 上传文档（PDF/Word/TXT）
4. 等待文档处理完成
5. 在工作流中选择知识检索节点并配置知识库

## 七、项目评价

### 7.1 优势

- **代码结构清晰**：模块化设计，易于理解和扩展
- **技术栈现代**：Vue3 + NestJS 是当前主流选择
- **功能完整可用**：基础功能都已实现，可以正常运行
- **适合学习**：代码质量高，适合作为学习项目

### 7.2 局限

- **工作流引擎简单**：缺少循环、并行等复杂控制流
- **单用户架构**：无多租户、权限管理
- **向量存储有限**：SQLite 不适合大规模向量检索
- **缺少运维功能**：无监控统计、日志审计

### 7.3 适用场景

- ✅ **学习目的**：理解工作流引擎和 RAG 原理
- ✅ **个人使用**：搭建个人 AI 助手
- ✅ **快速原型**：验证产品想法
- ❌ **企业生产**：缺少必要的运维和权限功能

## 八、扩展建议

如果要继续完善，建议按以下优先级：

1. **HTTP请求节点** - 让工作流能调用外部API（高优先级）
2. **变量管理** - 临时存储和传递数据（高优先级）
3. **循环节点** - 批量处理能力（中优先级）
4. **用户系统** - 多用户支持（中优先级）
5. **统计监控** - 用量和性能监控（低优先级）
