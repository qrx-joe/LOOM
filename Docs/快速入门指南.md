# Mini-Coze 快速入门指南

## 🚀 项目简介

Mini-Coze 是一个可视化 AI 工作流编排工具，让你像搭积木一样构建 AI 应用。无需编程，通过拖拽节点即可实现智能问答、知识库检索等功能。

---

## 📋 环境准备

### 必要条件

- **Node.js** 18+ （推荐 20+）
- **npm** 或 **yarn**
- **Git**（可选）

### 检查环境

```bash
node --version    # 应该显示 v18.x.x 或更高
npm --version     # 应该显示 9.x.x 或更高
```

## 🛠️ 安装与启动

### 方式一：前后端分别启动

#### 1. 启动后端

```bash
# 进入后端目录
cd backend

# 安装依赖
npm install

# 启动开发服务器
npm run start:dev
```

后端启动成功后会显示：
```
[Nest] xxx  - LOG  [NestFactory] Starting Nest application...
[Nest] xxx  - LOG  [InstanceLoader] TypeOrmModule dependencies initialized
```

后端运行在：**http://localhost:3001**

#### 2. 启动前端

```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端启动成功后会显示：
```
VITE v7.3.1  ready in xxx ms
➜  Local:   http://localhost:5173/
```

前端运行在：**http://localhost:5173**（或 5174，如果 5173 被占用）

#### 3. 访问应用

打开浏览器，访问 **http://localhost:5173**

## 🎯 第一个工作流：智能问答

### 步骤 1：创建工作流

1. 在首页点击「新建工作流」
2. 输入名称：「我的第一个 AI 助手」
3. 点击「创建」

### 步骤 2：添加节点

从左侧工具栏拖拽以下节点到画布：

| 顺序 | 节点 | 作用 |
|------|------|------|
| 1 | 开始 | 接收用户输入 |
| 2 | AI 回答 | 调用大模型 |
| 3 | 结束 | 输出结果 |

### 步骤 3：连接节点

点击节点边缘的小圆点，拖拽到下一个节点：

```
开始 → AI 回答 → 结束
```

### 步骤 4：配置 AI 节点

1. 点击「AI 回答」节点
2. 在右侧面板配置：
   - **Prompt**：`请回答以下问题：{{START_INPUT}}`
   - **模型**：`deepseek-ai/DeepSeek-V3`
   - **温度**：`0.7`

### 步骤 5：运行测试

1. 点击顶部「运行」按钮
2. 在弹出的输入框中输入：「你好，请介绍一下自己」
3. 点击「确认」
4. 查看右侧「执行日志」面板，等待 AI 回复

## 📚 第二个工作流：知识库问答

### 步骤 1：创建知识库

1. 点击左侧菜单「知识库」
2. 点击「新建知识库」
3. 输入名称：「产品手册」
4. 点击「创建」

### 步骤 2：上传文档

1. 进入「产品手册」知识库
2. 点击「上传文档」
3. 选择 PDF 或 Word 文件
4. 等待处理完成（状态变为「已完成」）

### 步骤 3：创建工作流

1. 回到「工作流」页面
2. 点击「新建工作流」
3. 输入名称：「智能客服」

### 步骤 4：添加节点并连接

```
开始 → 知识检索 → AI 回答 → 结束
```

### 步骤 5：配置节点

**知识检索节点**：
- 知识库：选择「产品手册」
- 查询模板：`{{START_INPUT}}`

**AI 回答节点**：
- Prompt：
```
基于以下知识回答用户问题：
{{knowledge-1.output}}

用户问题：{{START_INPUT}}

请根据提供的知识给出准确回答。
```

### 步骤 6：运行测试

1. 点击「运行」
2. 输入与文档相关的问题
3. 查看 AI 基于文档的回答

## 🎨 节点类型说明

### 开始节点

- **作用**：工作流的入口，接收用户输入
- **配置**：无需特殊配置
- **输出**：用户输入的内容

### AI 回答节点

- **作用**：调用大语言模型生成回答
- **配置**：
  - **Prompt**：给 AI 的指令，支持变量 `{{START_INPUT}}`、`{{nodeId.output}}`
  - **模型**：选择使用的大模型
  - **温度**：控制回答的创造性（0-1，越大越随机）
- **输出**：`{ text: "AI生成的回答" }`

### 知识检索节点

- **作用**：从知识库搜索相关文档片段
- **配置**：
  - **知识库**：选择要检索的知识库
  - **查询模板**：搜索关键词，支持变量
- **输出**：`{ fragments: [{ content, score, documentName }] }`

### 条件节点

- **作用**：根据条件走不同分支
- **配置**：
  - **条件表达式**：如 `{{input}} > 10`
- **输出**：`{ conditionResult: "true" 或 "false" }`

### 结束节点

- **作用**：工作流的出口，返回最终结果
- **配置**：无需特殊配置

## 🔧 常用操作

### 撤销/重做

- **Ctrl + Z**：撤销上一步
- **Ctrl + Shift + Z**：重做
- 最多保存 50 步历史

### 保存工作流

- 点击「保存」按钮或按 **Ctrl + S**
- 工作流会自动保存到数据库

### 删除节点

1. 点击选中节点
2. 按 **Delete** 键，或右键选择「删除」

### 查看执行日志

1. 运行工作流后，点击「日志」面板
2. 查看每个节点的执行状态和输出

### 查看节点结果

1. 运行工作流后
2. **双击**节点
3. 查看该节点的输入、输出和配置

## 💡 最佳实践

### Prompt 编写技巧

1. **明确指令**：告诉 AI 具体要做什么
   ```
   ❌ 不好：回答问题
   ✅ 好：你是一个专业客服，请基于以下知识回答用户问题
   ```

2. **使用变量**：动态注入上下文
   ```
   知识：{{knowledge-1.output}}
   问题：{{START_INPUT}}
   ```

3. **设置角色**：让 AI 扮演特定角色
   ```
   你是一位经验丰富的医生，请根据症状给出建议...
   ```

### 知识库优化

1. **文档质量**：上传结构清晰、内容完整的文档
2. **分片大小**：默认 1000 字符，可根据内容调整
3. **检索数量**：合理设置返回的片段数量（默认 top 5）

### 工作流设计

1. **保持简单**：一个工作流只做一件事
2. **模块化**：复杂逻辑拆分成多个子工作流（待实现）
3. **充分测试**：运行多种输入测试边界情况

## ❓ 常见问题

### Q1: 前端无法启动，提示端口被占用

**解决**：
```bash
# 方法1：关闭占用 5173 端口的程序
# 方法2：前端会自动尝试下一个端口（5174）
```

### Q2: 后端无法连接数据库

**解决**：
```bash
# 检查是否有其他程序占用了 SQLite 数据库文件
# 删除 mini-coze.db 重新启动（会丢失数据）
```

### Q3: AI 节点运行失败

**排查**：
1. 检查是否配置了 API Key
2. 检查网络连接
3. 查看日志中的错误信息

### Q4: 知识检索返回空结果

**排查**：
1. 检查知识库是否有文档
2. 检查文档是否处理完成
3. 尝试使用更通用的查询词

### Q5: 工作流运行卡住

**解决**：
1. 检查是否有循环依赖
2. 检查条件节点是否配置了分支
3. 刷新页面重新运行
