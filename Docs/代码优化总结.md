## 一、Bug 修复

### 1.1 空 catch 块

**问题**：`FlowCanvas.vue` 中的 SSE 消息处理使用了空 catch，导致错误被静默吞掉

**修复**：
```typescript
// 修复前
catch (e) {}

// 修复后
catch (e) {
  console.error('Failed to parse SSE message:', e, event.data)
}
```

### 1.2 SSE 内存泄漏

**问题**：组件卸载时未关闭 EventSource，导致内存泄漏

**修复**：
```typescript
// 组件级变量存储 EventSource
let currentEventSource: EventSource | null = null

// onUnmounted 时清理
onUnmounted(() => {
  if (currentEventSource) {
    currentEventSource.close()
    currentEventSource = null
  }
})
```

**学习收获**：
- 错误处理最佳实践
- Vue 组件生命周期管理
- 资源清理的重要性

## 三、类型统一

### 3.1 问题

前后端类型定义不一致，导致大量兼容代码：

```typescript
// 前端
const inputNodes = nodes.filter(n =>
  n.type === 'input' || n.type === 'START' || n.type === 'INPUT'
)

// 后端
const inputNodes = nodes.filter(n =>
  n.type === NodeType.INPUT || n.type === 'input'
)
```

### 3.2 解决方案

创建共享类型定义：

```typescript
// backend/src/common/types/workflow.types.ts
// frontend/src/types/workflow.types.ts

export enum NodeType {
  START = 'START',
  INPUT = 'INPUT',  // 兼容旧版本
  END = 'END',
  OUTPUT = 'OUTPUT',
  AI_AGENT = 'AI_AGENT',
  KNOWLEDGE_RETRIEVAL = 'KNOWLEDGE_RETRIEVAL',
  CONDITION = 'CONDITION',
}

export interface WorkflowNode {
  id: string
  type: string
  label: string
  position: { x: number; y: number }
  data?: NodeData
}
```

### 3.3 工具函数复用

```typescript
// 前后端共用
export function isStartNodeType(type: string): boolean {
  return type === NodeType.START || type === NodeType.INPUT
}
```

**学习收获**：
- 单一数据源（Single Source of Truth）
- DRY 原则（Don't Repeat Yourself）
- TypeScript 类型系统

## 四、策略模式重构条件节点

### 4.1 问题

原 `handleCondition` 方法使用大量 if-else：

```typescript
// 35 行复杂代码
const matchGt = expression.match(/{{(\w+)}}\s*>\s*(\d+)/);
const matchLt = expression.match(/{{(\w+)}}\s*<\s*(\d+)/);
const matchEq = expression.match(/{{(\w+)}}\s*==\s*"?([^"]+)"?/);

if (matchGt) {
  // ...
} else if (matchLt) {
  // ...
} else if (matchEq) {
  // ...
}
```

**问题**：
- 违反开闭原则（新增条件类型需修改原有代码）
- 难以测试
- 可读性差

### 4.2 策略模式实现

**核心思想**：定义一系列算法，将它们封装起来，并且使它们可以互相替换。

```typescript
// 策略接口
interface ConditionStrategy {
  readonly name: string;
  matches(expression: string): boolean;
  evaluate(input: any, expression: string): boolean;
}

// 具体策略
class GreaterThanStrategy implements ConditionStrategy {
  readonly name = 'greaterThan';
  matches(expr) { return />/.test(expr) && !/>=/.test(expr); }
  evaluate(input, expr) { /* ... */ }
}

// 策略上下文
class ConditionContext {
  private strategies: ConditionStrategy[] = [];

  evaluate(input: any, expression: string) {
    const strategy = this.strategies.find(s => s.matches(expression));
    return strategy.evaluate(input, expression);
  }
}
```

### 4.3 支持的条件类型

| 策略 | 表达式示例 |
|------|-----------|
| greaterThan | `{{score}} > 10` |
| lessThan | `{{age}} < 18` |
| equal | `{{name}} == "John"` |
| notEqual | `{{status}} != "error"` |
| greaterThanOrEqual | `{{score}} >= 60` |
| lessThanOrEqual | `{{score}} <= 100` |
| contains | `{{content}} contains "keyword"` |
| startsWith | `{{name}} startsWith "Mr"` |
| endsWith | `{{email}} endsWith "@example.com"` |
| exists | `{{name}} exists` |

### 4.4 代码对比

**重构前**（35 行）：
```typescript
private async handleCondition(node, config, input) {
  const expression = config.expression || '';
  let conditionResult = 'true';

  try {
    const matchGt = expression.match(/.../);
    const matchLt = expression.match(/.../);
    const matchEq = expression.match(/.../);

    if (matchGt) {
      // ...
    } else if (matchLt) {
      // ...
    } else if (matchEq) {
      // ...
    }
  } catch (e) {
    this.logger.warn(`...`);
  }

  return { conditionResult, ...input };
}
```

**重构后**（10 行）：
```typescript
private async handleCondition(node, config, input) {
  const expression = config.expression || '';

  // 使用策略模式
  const { result, strategy } = this.conditionContext.evaluate(input, expression);
  const conditionResult = result ? 'true' : 'false';

  this.logger.log(`... using ${strategy} strategy`);
  return { conditionResult, ...input };
}
```
