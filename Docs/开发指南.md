# Mini-Coze 开发指南

## 一、开发环境搭建

### 1.1 前置要求

- Node.js 18+
- npm 9+ 或 yarn
- Git
- VS Code（推荐）

### 1.2 推荐 VS Code 插件

- **Volar** - Vue 3 官方插件
- **ESLint** - 代码检查
- **Prettier** - 代码格式化
- **TypeScript Importer** - 自动导入
- **Tailwind CSS IntelliSense** - CSS 提示

### 1.3 项目克隆与安装

```bash
# 克隆项目
git clone https://github.com/qrx-joe/mini-coze.git
cd mini-coze

# 安装后端依赖
cd backend
npm install

# 安装前端依赖
cd ../frontend
npm install
```

### 3.2 创建新模块步骤

以创建「日志模块」为例：

**步骤 1：创建目录结构**

```bash
mkdir -p backend/src/logs/{dto,entities,interfaces}
touch backend/src/logs/{logs.controller.ts,logs.service.ts,logs.module.ts,index.ts}
```

**步骤 2：定义实体**

```typescript
// logs/entities/log.entity.ts
import { Entity, PrimaryGeneratedColumn, Column, CreateDateColumn } from 'typeorm';

@Entity()
export class Log {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column()
  level: string;

  @Column('text')
  message: string;

  @CreateDateColumn()
  createdAt: Date;
}
```

**步骤 3：创建服务**

```typescript
// logs/logs.service.ts
import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { Log } from './entities/log.entity';

@Injectable()
export class LogsService {
  constructor(
    @InjectRepository(Log)
    private logRepo: Repository<Log>,
  ) {}

  async findAll() {
    return this.logRepo.find();
  }

  async create(level: string, message: string) {
    const log = this.logRepo.create({ level, message });
    return this.logRepo.save(log);
  }
}
```

**步骤 4：创建控制器**

```typescript
// logs/logs.controller.ts
import { Controller, Get, Post, Body } from '@nestjs/common';
import { LogsService } from './logs.service';

@Controller('logs')
export class LogsController {
  constructor(private readonly logsService: LogsService) {}

  @Get()
  findAll() {
    return this.logsService.findAll();
  }

  @Post()
  create(@Body() body: { level: string; message: string }) {
    return this.logsService.create(body.level, body.message);
  }
}
```

**步骤 5：创建模块**

```typescript
// logs/logs.module.ts
import { Module } from '@nestjs/common';
import { TypeOrmModule } from '@nestjs/typeorm';
import { LogsController } from './logs.controller';
import { LogsService } from './logs.service';
import { Log } from './entities/log.entity';

@Module({
  imports: [TypeOrmModule.forFeature([Log])],
  controllers: [LogsController],
  providers: [LogsService],
})
export class LogsModule {}
```

**步骤 6：注册模块**

```typescript
// app.module.ts
import { LogsModule } from './logs/logs.module';

@Module({
  imports: [
    // ... 其他模块
    LogsModule,
  ],
})
export class AppModule {}
```

### 3.3 工作流节点开发

**添加新节点类型的步骤：**

**步骤 1：在接口中定义节点类型**

```typescript
// workflow/interfaces/workflow.interface.ts
export enum NodeType {
  // ... 现有类型
  HTTP_REQUEST = 'HTTP_REQUEST',  // 新类型

```

**步骤 2：在执行引擎中处理节点**

```typescript
// workflow/executor.service.ts
private async executeNode(node: NodeData, input: any): Promise<any> {
  const nodeType = node.type as string;
  switch (nodeType) {
    // ... 现有 case
    case 'HTTP_REQUEST':
    case NodeType.HTTP_REQUEST:
      return this.handleHttpRequest(node, input);
  }
}

private async handleHttpRequest(node: NodeData, input: any): Promise<any> {
  const config = (node as any).data || {};
  const url = config.url;
  const method = config.method || 'GET';

  // 实现 HTTP 请求逻辑
  const response = await fetch(url, { method });
  const data = await response.json();

  return { data };
}
```

### 4.3 状态管理

**使用 Pinia：**

```typescript
// store/workflow.ts
import { defineStore } from 'pinia';
import { ref, computed } from 'vue';
import type { Node, Edge } from '@vue-flow/core';

export const useWorkflowStore = defineStore('workflow', () => {
  // State
  const nodes = ref<Node[]>([]);
  const edges = ref<Edge[]>([]);
  const currentWorkflow = ref<any>(null);

  // Getters
  const nodeCount = computed(() => nodes.value.length);
  const isEmpty = computed(() => nodes.value.length === 0);

  // Actions
  const addNode = (node: Node) => {
    nodes.value.push(node);
  };

  const removeNode = (nodeId: string) => {
    nodes.value = nodes.value.filter(n => n.id !== nodeId);
  };

  const updateNode = (nodeId: string, data: any) => {
    const node = nodes.value.find(n => n.id === nodeId);
    if (node) {
      Object.assign(node.data, data);
    }
  };

  return {
    nodes,
    edges,
    currentWorkflow,
    nodeCount,
    isEmpty,
    addNode,
    removeNode,
    updateNode,
  };
});
```

### 4.4 API 调用封装

const api = axios.create({
  baseURL: API_BASE_URL,

export const workflowApi = {

  getById: (id: string) => api.get(`/workflows/${id}`),
  create: (data: any) => api.post('/workflows', data),
  update: (id: string, data: any) => api.put(`/workflows/${id}`, data),
  delete: (id: string) => api.delete(`/workflows/${id}`),
  run: (id: string, input: any) => api.post(`/workflows/${id}/run`, { input }),
};

## 六、常见问题

### Q1: 数据库变更后不生效

**解决：**
```bash
# 删除数据库文件重新生成
cd backend
rm mini-coze.db
npm run start:dev
```

### Q2: 热重载失效

**解决：**
```bash
# 后端
npm run start:dev

# 前端
npm run dev
```

### Q3: TypeScript 类型错误

**解决：**
```bash
# 重新安装类型定义
npm install

# 或者清除缓存
rm -rf node_modules package-lock.json
npm install
```

### 7.1 提交规范

使用 Conventional Commits：

```
feat: 添加新功能
fix: 修复 bug
docs: 文档更新
style: 代码格式调整
refactor: 重构
perf: 性能优化
test: 测试相关
chore: 构建/工具相关
```

示例：
```bash
git commit -m "feat: 添加 HTTP 请求节点"
git commit -m "fix: 修复知识检索结果为空的问题"
```

- 使用 ESLint 和 Prettier
- 变量名使用 camelCase
- 常量名使用 UPPER_SNAKE_CASE
- 类型名使用 PascalCase
